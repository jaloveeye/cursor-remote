## 테스트 개요

### 목적
Cursor 2.4의 새로운 기능들(Skills, Subagents, Image Generation, Clarification Questions)이 Cursor Remote 프로젝트에서 올바르게 작동하는지 검증합니다.

### 사전 요구사항
- Cursor IDE 2.4 이상 설치
- Cursor Remote 프로젝트가 로컬에 클론됨
- `SKILL.md`, `.cursor/subagents.md`, `assets/` 디렉토리 존재 확인

### 테스트 환경
- **OS**: macOS, Windows, Linux
- **Cursor 버전**: 2.4 이상
- **프로젝트**: cursor-remote (develop 브랜치)

## Skills 기능 테스트

### 테스트 목표
`SKILL.md` 파일이 Cursor의 Skills 기능으로 올바르게 인식되고 활용되는지 확인합니다.

### 테스트 시나리오

#### 시나리오 1: Skills 자동 발견
**목적**: Cursor가 `SKILL.md` 파일을 자동으로 발견하는지 확인

**절차**:
1. Cursor IDE에서 프로젝트 열기
2. 새 채팅 시작
3. 슬래시 명령 메뉴 열기 (`/` 입력)

**예상 결과**:
- 슬래시 명령 메뉴에 Skills 관련 옵션이 표시됨
- 또는 에이전트가 Skills를 자동으로 인식

**검증 방법**:
- [ ] 슬래시 명령 메뉴에 Skills 옵션 확인
- [ ] 에이전트가 Skills를 언급하는지 확인

#### 시나리오 2: 세션 관리 Skill 사용
**목적**: 세션 관리 워크플로우가 Skill로 올바르게 작동하는지 확인

**절차**:
1. 채팅에서 다음 요청:
   ```
   "세션 정보를 조회해줘"
   ```
2. 또는 슬래시 명령 사용:
   ```
   /session-info
   ```

**예상 결과**:
- 에이전트가 `SKILL.md`의 세션 관리 섹션을 참조
- 올바른 파일(`cursor-extension/src/cli-handler.ts`)을 확인
- 세션 조회 방법을 제안

**검증 방법**:
- [ ] 에이전트가 `SKILL.md`를 참조하는지 확인
- [ ] 올바른 파일 경로를 제시하는지 확인
- [ ] 워크플로우가 정확한지 확인

#### 시나리오 3: 대화 히스토리 Skill 사용
**목적**: 대화 히스토리 조회 워크플로우가 Skill로 작동하는지 확인

**절차**:
1. 채팅에서 다음 요청:
   ```
   "이전 대화 히스토리를 조회하는 방법을 알려줘"
   ```

**예상 결과**:
- 에이전트가 `SKILL.md`의 대화 히스토리 섹션 참조
- `.cursor/CHAT_HISTORY.json` 파일 위치 설명
- 히스토리 조회 방법 제안

**검증 방법**:
- [ ] `SKILL.md`의 히스토리 섹션 참조 확인
- [ ] 파일 경로가 정확한지 확인
- [ ] 데이터 구조 설명이 정확한지 확인

#### 시나리오 4: Extension 개발 Skill 사용
**목적**: Extension 개발 워크플로우가 Skill로 작동하는지 확인

**절차**:
1. 채팅에서 다음 요청:
   ```
   "Extension에 새로운 기능을 추가하고 싶어"
   ```

**예상 결과**:
- 에이전트가 `SKILL.md`의 Extension 개발 섹션 참조
- 프로젝트 구조 설명
- 개발 워크플로우 제안

**검증 방법**:
- [ ] Extension 개발 섹션 참조 확인
- [ ] 프로젝트 구조 설명이 정확한지 확인
- [ ] 컴파일 및 테스트 방법 제안 확인

## Subagents 기능 테스트

### 테스트 목표
`.cursor/subagents.md` 파일이 서브 에이전트로 올바르게 작동하는지 확인합니다.

### 테스트 시나리오

#### 시나리오 1: Subagents 자동 인식
**목적**: Cursor가 서브 에이전트 설정을 자동으로 인식하는지 확인

**절차**:
1. Cursor IDE에서 프로젝트 열기
2. Plan 모드로 복잡한 작업 요청:
   ```
   "세션 관리 기능을 개선하고, 모바일 앱 UI도 업데이트하고, 테스트도 추가해줘"
   ```

**예상 결과**:
- Cursor가 여러 서브 에이전트를 자동으로 생성
- Extension Development Agent, Flutter App Development Agent, Testing Agent 등이 병렬로 작업

**검증 방법**:
- [ ] 서브 에이전트가 자동으로 생성되는지 확인
- [ ] 각 에이전트가 올바른 파일에 접근하는지 확인
- [ ] 병렬 작업이 진행되는지 확인

#### 시나리오 2: Extension Development Agent
**목적**: Extension 전용 서브 에이전트가 올바르게 작동하는지 확인

**절차**:
1. Plan 모드에서:
   ```
   "Extension에 세션 히스토리 내보내기 기능을 추가해줘"
   ```
2. 서브 에이전트가 Extension 관련 파일만 수정하는지 확인

**예상 결과**:
- Extension Development Agent가 생성됨
- `cursor-extension/src/**/*.ts` 파일만 수정
- 다른 컴포넌트 파일은 수정하지 않음

**검증 방법**:
- [ ] Extension 파일만 수정되는지 확인
- [ ] Flutter 앱이나 PC 서버 파일은 수정되지 않는지 확인
- [ ] TypeScript/VS Code Extension API를 올바르게 사용하는지 확인

#### 시나리오 3: Flutter App Development Agent
**목적**: Flutter 전용 서브 에이전트가 올바르게 작동하는지 확인

**절차**:
1. Plan 모드에서:
   ```
   "모바일 앱에 세션 히스토리 내보내기 UI를 추가해줘"
   ```
2. 서브 에이전트가 Flutter 파일만 수정하는지 확인

**예상 결과**:
- Flutter App Development Agent가 생성됨
- `mobile-app/lib/**/*.dart` 파일만 수정
- Flutter/Dart 베스트 프랙티스 준수

**검증 방법**:
- [ ] Flutter 파일만 수정되는지 확인
- [ ] Extension이나 PC 서버 파일은 수정되지 않는지 확인
- [ ] Material Design 가이드라인 준수 확인

#### 시나리오 4: 병렬 작업 테스트
**목적**: 여러 서브 에이전트가 병렬로 작업하는지 확인

**절차**:
1. Plan 모드에서:
   ```
   "Extension에 API를 추가하고, 모바일 앱에 UI를 추가하고, PC 서버에 라우팅을 추가해줘"
   ```

**예상 결과**:
- 3개의 서브 에이전트가 동시에 작업 시작
- 각 에이전트가 독립적으로 작업
- 작업 완료 후 결과 통합

**검증 방법**:
- [ ] 여러 에이전트가 동시에 작업하는지 확인
- [ ] 각 에이전트의 작업이 독립적인지 확인
- [ ] 최종 결과가 올바르게 통합되는지 확인

## Image Generation 기능 테스트

### 테스트 목표
Cursor의 이미지 생성 기능이 올바르게 작동하고 `assets/` 디렉토리에 저장되는지 확인합니다.

### 테스트 시나리오

#### 시나리오 1: 아키텍처 다이어그램 생성
**목적**: 시스템 아키텍처 다이어그램을 생성하는지 확인

**절차**:
1. 채팅에서 다음 요청:
   ```
   "Cursor Remote 시스템의 아키텍처 다이어그램을 생성해줘. 
   Mobile App (Flutter), PC Server (Node.js), Extension (TypeScript), 
   Relay Server (Vercel), Cursor CLI를 포함해서"
   ```
2. 또는 슬래시 명령:
   ```
   /image "Create architecture diagram for Cursor Remote system"
   ```

**예상 결과**:
- 이미지가 생성됨
- 인라인 미리보기 표시
- `assets/architecture-*.png` 파일로 저장

**검증 방법**:
- [ ] 이미지가 생성되는지 확인
- [ ] 미리보기가 표시되는지 확인
- [ ] `assets/` 디렉토리에 파일이 저장되는지 확인
- [ ] 파일명이 적절한지 확인 (`architecture-*.png`)

#### 시나리오 2: UI 목업 생성
**목적**: 모바일 앱 UI 목업을 생성하는지 확인

**절차**:
1. 채팅에서 다음 요청:
   ```
   "모바일 앱의 세션 관리 화면 UI 목업을 생성해줘"
   ```
2. 또는 슬래시 명령:
   ```
   /image "Create UI mockup for mobile app session management screen"
   ```

**예상 결과**:
- UI 목업 이미지 생성
- `assets/ui-mockup-*.png` 파일로 저장

**검증 방법**:
- [ ] UI 목업이 생성되는지 확인
- [ ] `assets/` 디렉토리에 저장되는지 확인
- [ ] 파일명이 적절한지 확인 (`ui-mockup-*.png`)

#### 시나리오 3: 참조 이미지 업로드
**목적**: 참조 이미지를 업로드하여 생성하는지 확인

**절차**:
1. 채팅에서 이미지 파일 업로드
2. 다음 요청:
   ```
   "이 이미지를 참고해서 Cursor Remote 로고를 생성해줘"
   ```

**예상 결과**:
- 참조 이미지를 기반으로 새 이미지 생성
- `assets/asset-*.png` 파일로 저장

**검증 방법**:
- [ ] 참조 이미지가 인식되는지 확인
- [ ] 참조를 반영한 이미지가 생성되는지 확인
- [ ] `assets/` 디렉토리에 저장되는지 확인

#### 시나리오 4: Git 무시 확인
**목적**: 생성된 이미지가 Git에 커밋되지 않는지 확인

**절차**:
1. 이미지 생성 후 `git status` 실행
2. `assets/*.png` 파일이 추적되지 않는지 확인

**예상 결과**:
- `assets/*.png`, `assets/*.jpg` 파일이 `.gitignore`에 의해 무시됨
- `git status`에 표시되지 않음

**검증 방법**:
- [ ] `.gitignore`에 이미지 파일 패턴이 있는지 확인
- [ ] `git status`에 이미지 파일이 표시되지 않는지 확인

## Clarification Questions 기능 테스트

### 테스트 목표
에이전트가 명확화 질문을 올바르게 사용하는지 확인합니다.

### 테스트 시나리오

#### 시나리오 1: 자동 질문
**목적**: 불명확한 요구사항에 대해 자동으로 질문하는지 확인

**절차**:
1. 채팅에서 모호한 요청:
   ```
   "세션 관리 기능을 개선해줘"
   ```

**예상 결과**:
- 에이전트가 자동으로 질문:
  - "어떤 부분을 개선하고 싶으신가요?"
  - "세션 생성 속도? 세션 격리 정확도? 세션 히스토리 관리?"

**검증 방법**:
- [ ] 에이전트가 질문을 하는지 확인
- [ ] 질문이 구체적이고 선택지가 제공되는지 확인
- [ ] `SKILL.md`의 질문 예시와 유사한지 확인

#### 시나리오 2: 기술 선택 질문
**목적**: 기술 선택이 필요한 경우 질문하는지 확인

**절차**:
1. 채팅에서:
   ```
   "새로운 기능을 추가하고 싶어"
   ```

**예상 결과**:
- 에이전트가 질문:
  - "어떤 방식으로 구현하시겠습니까?"
  - "Extension에 추가? PC 서버에 추가? 모바일 앱에 추가?"

**검증 방법**:
- [ ] 기술 선택 질문이 나오는지 확인
- [ ] 여러 옵션이 제시되는지 확인
- [ ] 각 옵션의 장단점이 설명되는지 확인

#### 시나리오 3: 질문 대기 중 작업 계속
**목적**: 질문 대기 중에도 다른 작업을 계속하는지 확인

**절차**:
1. 모호한 요청 후 질문이 나옴
2. 질문에 답변하기 전에 에이전트가 다른 작업을 하는지 확인

**예상 결과**:
- 질문 대기 중에도 파일 읽기, 편집, 명령 실행 계속
- 답변 수신 즉시 작업에 반영

**검증 방법**:
- [ ] 질문 후에도 작업이 계속되는지 확인
- [ ] 답변 수신 시 즉시 반영되는지 확인

#### 시나리오 4: 서브 에이전트 질문
**목적**: 서브 에이전트가 질문 도구를 사용하는지 확인

**절차**:
1. Plan 모드에서:
   ```
   "Extension에 기능을 추가해줘"
   ```
2. Extension Development Agent가 질문하는지 확인

**예상 결과**:
- Extension Development Agent가 질문:
  - "어떤 파일을 수정해야 하나요?"
  - "예상되는 동작은 무엇인가요?"
  - "제약사항이나 엣지 케이스가 있나요?"

**검증 방법**:
- [ ] 서브 에이전트가 질문하는지 확인
- [ ] `.cursor/subagents.md`의 질문 예시와 유사한지 확인
- [ ] 질문이 해당 에이전트의 전문 분야와 관련있는지 확인

#### 시나리오 5: 명시적 질문 요청
**목적**: `/ask` 또는 `/clarify` 명령이 작동하는지 확인

**절차**:
1. 채팅에서:
   ```
   /ask "현재 작업에 대해 질문이 있어"
   ```
2. 또는:
   ```
   /clarify
   ```

**예상 결과**:
- 에이전트가 현재 작업에 대한 명확화 질문

**검증 방법**:
- [ ] 명령이 인식되는지 확인
- [ ] 적절한 질문이 나오는지 확인
